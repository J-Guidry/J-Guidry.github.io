Writeup

From this one, I reinforced the basics I learned earlier. Objects to organize code. Methods for reusable code. Separating data and UI code.

Using local storage for persistance, the todolist app uses object literals to organize code. The first one, labelled TodoList, has methods associated with storage, adding, editing, deleting, and toggling todo items. The second one is for the UI. The third is for the interaction of the two. It event listeners and an init function to set up the event listeners.

This uses Font Awesome for icons such as the circle, the X, and checkmark. Editing is done by double clicking a list item. Escape key will cancel and turn it back into a list item. Enter will save it. Clicking on the circle will toggle the item and make a strikethrough with opacity and change the circle to a check.
